# Button → GA4 Event Map

Every tracked button pushes a `button_event` value to the dataLayer; GTM's
"GA4 Event - per button" tag sends it as the GA4 **event name**, so each
button instance appears as its own row in GA4's Events report.

**Format:** `<location>_<intent>_click` — max 40 chars (GA4 hard limit).
Shared components (header, header_mobile, CTABanner final_cta) are prefixed
with the page slug, so the same button on different pages stays distinct.
Generated by the logic in `site/lib/analytics.ts` (`buttonEvent()`).

- Semantic events (`start_trial`, `get_free_score`, `book_demo`, `pricing_click`, `cta_click`) still fire in parallel for KPI rollups and key events.
- Header/CTABanner rows above show one example page; the page-slug prefix changes with the page (e.g. `pricing_header_login_click`, `customers_header_login_click`).
- Names longer than 40 chars are truncated by `buttonEvent()`; current inventory has none.
- Talk-to-Sales/Expert buttons fire ONLY `book_demo` (the cta_click double-fire was removed) so each physical click = one per-button event.
